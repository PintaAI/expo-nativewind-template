# Expo HAS CHANGED

Read the exact versioned docs at https://docs.expo.dev/versions/v56.0.0/ before writing any code.

## Template Identity

- **Template package:** `expo-nativewind-template`
- **Default app name:** `Expo NativeWind Starter`
- **Default slug/scheme:** `expo-nativewind-starter`
- **Default bundle ID:** `com.example.app`
- **App Version:** `1.0.0` with `runtimeVersion.policy = appVersion`
- **EAS Channels:** `development`, `preview`, `production` mapped in `eas.json`
- Run `npm run init-template` in a new app to replace identity values and optionally add EAS owner/project ID.

## Project Setup

- Expo SDK 56 with Expo Router, React 19.2.3, React Native 0.85.3, TypeScript 6.0.
- NativeWind v5 preview with Tailwind CSS v4 and `react-native-css` v3, using CSS-based config without a Babel transform.
- Dev-client mode is the default local development target.
- Tunnel mode is default for native dev server; `@expo/ngrok` is installed locally.
- `lightningcss` is pinned to `1.30.1` via overrides for Tailwind/NativeWind CSS processing.
- `src/global.css` imports Tailwind and NativeWind theme support.
- `expo-router` entry point via `"main": "expo-router/entry"`.
- `reactCompiler` and typed routes are enabled in `app.json`.
- `app.json` plugins: `expo-router`, `expo-splash-screen`.
- `@expo/ui` components are available.

## Architecture

```text
src/
├── app/
│   ├── _layout.tsx              # Root providers and native tabs
│   ├── (home)/                  # Home tab with drawer and nested stack
│   └── settings/                # Settings tab with stack/detail screen
├── components/                  # Tabs, theme context, theme bridge
├── config/app.ts                # Template display metadata
└── global.css                   # Tailwind v4 and NativeWind theme CSS
```

- No native code directories (`ios/`, `android/`) are committed; they are generated by Expo prebuild and gitignored.
- Starter routes include home, settings, settings detail, and Expo Router's generated sitemap route.

## Common Commands

- `npm run init-template` initializes a new app from the template.
- `npm run dev` starts the Expo dev server in dev-client mode with tunnel and cache clear.
- `npm run android` starts dev-client mode with tunnel and opens Android.
- `npm run ios` starts dev-client mode with tunnel and opens iOS.
- `npm run build:android` builds a local Android dev-client APK.
- `npm run build:android:preview` builds a local Android preview APK.
- `npm run build:ios` starts an EAS cloud iOS development build.
- `npm run run:android` uses `expo run:android` for local native Android install/run.
- `npm run web` starts web-only Expo dev server.
- `npm run lint` runs `expo lint`.

## TypeScript

- **Config:** Extends `expo/tsconfig.base`, `strict: true`
- **Path Aliases:** `@/*` -> `./src/*`, `@/assets/*` -> `./assets/*`
- **Includes:** `.expo/types/**/*.ts`, `expo-env.d.ts`, `nativewind-env.d.ts`
- TypeScript 6.0 may have breaking changes from TS 5.x; check compatibility when adding tooling.

## NativeWind / Tailwind v4 Setup

- **Pipeline:** `postcss.config.mjs` -> `@tailwindcss/postcss` | `metro.config.js` -> `withNativewind(config)` | `nativewind-env.d.ts` -> `react-native-css/types`
- **No `babel.config.js`:** Expo SDK 56 handles Babel internally via Expo Router entry.
- **No `tailwind.config.*`:** Tailwind v4 uses CSS-based config via `@theme` directives.
- Theme tokens live in `src/global.css` and runtime values are provided by `VariableContextProvider` in `src/components/ThemeProvider.tsx`.

## Verification

- Prefer `npx tsc --noEmit` for TypeScript verification.
- Use `npx expo config --json` after app config, EAS, OTA, or plugin changes.
- Use `npx expo export --platform web --output-dir /tmp/expo-template-test-build` only as a non-interactive bundling check, not as a web-only assumption.

## Environment
